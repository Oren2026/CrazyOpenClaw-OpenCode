## 1. 模块定义

### 1.1 Oren（协调者）

**职责：**
- 接收黑皮的需求
- 分析任务复杂度
- 生成指令给 OpenCode
- 审核结果并决定是否需要修改

**决策树：**
```
黑皮的需求
    │
    ├─ 资讯查询 → Oren 直接回答
    │
    ├─ 简单代码 → Oren 自己写
    │
    └─ 复杂代码 → 调用 OpenCode
            │
            ├─ 单文件 → 直接下指令
            └─ 多文件 → 先写 SPEC.md 再执行
```

### 1.2 Prompt Bridge（指令桥）

**输入：** Oren 的任务描述
**输出：** 格式化 prompt

**Prompt 结构：**
```markdown
## 任务
[清晰描述要做什么]

## 文件
- 文件路径: 描述
- ...

## 约束
- 技术限制
- 风格限制

## 产出
- 期望结果

## 验证
- [ ] 检查点 1
- [ ] 检查点 2
```

### 1.3 OpenCode Agent

**角色：** AI Coding Agent
**约束：**
- 使用 /tmp/CrazyOpenClaw-OpenCode 为工作目录
- 所有产出 push 到 GitHub main 分支
- 完成后报告完成状态

## 2. 数据结构

### 2.1 任务对象

```typescript
interface Task {
  id: string;              // UUID
  type: 'simple' | 'feature' | 'bug';
  description: string;
  files: File[];
  constraints: string[];
  due?: Date;
}

interface File {
  path: string;
  description: string;
}

interface Result {
  taskId: string;
  status: 'done' | 'error' | 'partial';
  commitHash: string;
  files: string[];
  notes?: string;
}
```

## 3. API 定义

### 3.1 任务提交

```
POST /task
Content-Type: application/json

{
  "type": "feature",
  "description": "添加用户登录功能",
  "files": [
    {"path": "login.html", "description": "登录页面"},
    {"path": "auth.js", "description": "认证逻辑"}
  ],
  "constraints": ["无外部依赖", "原生JS"]
}
```

### 3.2 结果回调

```
POST /result
{
  "status": "done",
  "commit": "abc123",
  "files": ["login.html", "auth.js"],
  "notes": "..."
}
```

## 4. 错误处理

| 错误码 | 描述 | 处理方式 |
|--------|------|--------|
| E001   | OpenCode 超时 | 重试一次，报告错误 |
| E002   | 产出不符 | 重新下指令 |
| E003   | Git 冲突 | 合并后重试 |
| E004   | 技术不可行 | 回报黑皮，询问替代方案 |

## 5. 命名规范

### 分支命名
```
main              # 主分支
feature/[name]     # 功能分支
fix/[name]        # 修复分支
```

### 提交信息
```
feat:     # 新功能
fix:      # Bug 修复
docs:     # 文档
refactor: # 重构
chore:    # 杂项
```

## 6. 质量保证

### 代码审查清单
- [ ] 语法正确
- [ ] 可运行（无明显错误）
- [ ] 符合约束条件
- [ ] 已 push 到 GitHub

### Oren 审核要点
1. 代码是否符合需求？
2. 有没有安全风险？
3. 变量/函数命名是否清晰？
4. 是否需要文档？
